# MatrixEcon 项目规则

## PDF 论文自动翻译

- 当用户上传、指定或放入新的学术论文 PDF 时，默认将其视为需要进入 MatrixEcon 的论文处理任务；除非用户明确说只存档、只摘要或不翻译，否则自动启动全文翻译流程。
- 必须调用 `matrixecon-paper-workflow` 统筹材料盘点、翻译、文献解读、复现代码审计、图表映射、复现报告和网站接入。
- 翻译必须调用 `baoyu-translate`，并同时应用 `academic-translation-guide` 的学术术语、忠实性和译后审校规范。
- 使用项目配置 `.baoyu-skills/baoyu-translate/EXTEND.md`，默认输出简体中文，采用 `refined`、`academic` 模式。
- 翻译范围包括正文、脚注、图表标题与注释、正式附录、网络附录和补充材料。参考文献保留原文，必要时仅翻译文献标题。
- 脚注不得按 PDF 页底提取顺序直接混入正文。必须先完整恢复脚注锚点所在的正文段落，再将完整脚注插入该段落之后；跨页脚注应先合并全部内容。网站使用“〔脚注 N〕”锚点和紧随段落的独立脚注块。
- 译后审校必须检查：正文是否被页底脚注截断、脚注是否被拆成多段、脚注编号是否与正文锚点一致，以及脚注后正文是否从正确位置续接。
- 默认生成纯中文逐句或逐段译稿，不在网站正文展示英文原句或“原文第 X 页”标题；不得省略正文，不得将全文翻译替换为摘要。
- 保留章节层级、公式、变量名、引文、表格编号、图号和内部页码锚点。公式、代码、数据字段和文献引用不得擅自翻译或改写。
- 正文图表必须直接插入对应中文译文位置，不得只提供图表超链接。
- 长文必须分块处理，先建立全文术语表，再翻译各块，最后统一检查术语、数字、否定词、因果表述和跨块衔接。
- 最终至少生成可供网站读取的 Markdown 文件，并将全文译稿、论文解读和复现报告接入对应的论文详情页。
- 不得在未获用户明确授权时，将论文全文发送到 Google Translate、DeepL、Azure、百度翻译或其他第三方翻译服务。默认由当前 Codex 模型完成翻译。
- 如 PDF 存在扫描页、不可提取文本、公式识别失败或附录缺失，应先执行 OCR/材料盘点并明确记录，不得静默跳过。

## MatrixEcon 标准交付

- 论文详情页固定包含：中文全文翻译、研究问题、核心贡献、理论机制、数据与样本、识别策略、主要结果图、局限与争议、个人解读、复现报告。
- 理论机制部分不得虚构原文没有提出的结构模型。没有正式模型时，使用机制图和标明性质的概念简式。
- 数据与样本部分必须解释论文特有指标的“原始数据 → 清洗/缩放 → 分组 → 计算 → 经济含义”。
- 识别策略必须写明分析层级、处理变量、因变量、固定效应、聚类层级、识别假设和因果解释强度。
- 每个核心 Figure/Table 必须映射到生成代码、输入数据、关键变量构造和验证状态。
- 复现状态必须区分材料盘点、结构审计、作者预生成结果核对、伪数据运行和真实数据复现。没有真实运行证据时不得声称“复现成功”。
- 网站接入后必须完成内容核对、lint/类型检查/构建和 Browser 桌面端与 390px 移动端验证。
